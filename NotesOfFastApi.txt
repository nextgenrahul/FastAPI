uvicorn main:app --reload -> This command is used to run a FastAPI application using Uvicorn, which is an ASGI server. The `main:app` part indicates that the FastAPI application instance is located in the `main.py` file and is named `app`. The `--reload` flag allows the server to automatically reload when code changes are detected, which is useful during development.

* Order matter in FastAPI, the order of the routes and middleware can affect how requests are processed. For example, if you have a route that matches a certain path and is defined before another route that also matches that path, the first route will be executed, and the second one will be ignored. Similarly, if you have middleware that processes requests before they reach the routes, the order in which you define the middleware can affect how requests are handled. Therefore, it's important to carefully consider the order of your routes and middleware when building a FastAPI application.



# Schema Models : Schema models in FastAPI are used to define the structure of the data that is expected in requests and responses. They are typically defined using Pydantic, which is a data validation library. Schema models allow you to specify the fields, their types, and any validation rules for the data. This helps ensure that the data being sent to and from your API is consistent and meets the expected format. You can use schema models to define request bodies, query parameters, and response models in your FastAPI application.

# SqlAlchemy : SQLAlchemy is an Object-Relational Mapping (ORM) library for Python that provides a high-level interface for working with databases. It allows you to define your database schema using Python classes and provides tools for querying and manipulating the database using Python code. SQLAlchemy supports multiple database backends, including SQLite, PostgreSQL, MySQL, and more. It also provides features such as connection pooling, transaction management, and support for complex queries. In FastAPI, you can use SQLAlchemy to interact with your database and perform CRUD operations on your data models.
